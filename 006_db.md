# 006 – Database Design

**Database:** PostgreSQL 15  
**ORM:** SQLAlchemy 2.0 (async)  
**Migrations:** Alembic  

---

## Entity Relationship Diagram

```
channels ──< videos ──< video_metrics
                  └──< comments
                  └──< rankings

topics (standalone, linked to videos via topic_id in video_metrics in Phase 2)
```

### `rankings`
Stores the daily output for each ranking type.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | SERIAL | PK | |
| `type` | VARCHAR(20) | NOT NULL | buzz / discussion / viral |
| `video_id` | VARCHAR(20) | FK → videos.video_id | |
| `rank` | INTEGER | NOT NULL | 1-based |
| `score` | FLOAT | | The primary score for this type |
| `ranked_at` | TIMESTAMPTZ | DEFAULT NOW() | Partition key for daily snapshots |

**Composite index:** `(type, ranked_at DESC, rank)`

---

## Notes

- Old rankings are retained (not deleted). Query the latest `ranked_at` per type.
- `video_metrics` uses `video_id` as PK for fast upsert (`ON CONFLICT DO UPDATE`).
- `tags` and `keywords` stored as PostgreSQL native arrays for efficient overlap queries.
- All timestamps stored in UTC (`TIMESTAMPTZ`).
